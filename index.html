실시간 협업 화이트보드

여러 명이 같은 캔버스에서 동시에 도형을 그리고 즉시 동기화되는 웹 기반 협업 도구

약 814줄

Figma·Miro와 유사한 구조로 설계된 풀스택 웹 애플리케이션으로, Socket.IO를 통해 사각형·원 도형의 추가·이동·삭제를 실시간으로 모든 참여자에게 동기화합니다. URL 기반 방(room) 단위로 세션을 격리하며, 신규 접속자는 canvas:sync 스냅샷을 통해 현재 캔버스 전체 상태를 즉시 수신합니다. node:sqlite 영속화로 서버 재시작 후에도 데이터가 복원됩니다.

핵심 기능

  • 도형(사각형·원) 추가·이동·삭제 실시간 동기화 (Socket.IO)
  • URL 기반 방(room) 격리로 독립 세션 운영
  • 신규 접속 시 canvas:sync 스냅샷으로 전체 상태 일관성 보장
  • node:sqlite 영속화 — 서버 재시작 후 캔버스 복원
  • 동시 편집 충돌 처리 — liveMove(드래그 중 전파)·dragEnd(확정) 분리, last-write-wins 전략
  • 실시간 커서 추적 (40ms 스로틀) + 접속자 presence + 색상 자동 배정
  • 자동 재연결 및 끊김 상태 배너
  • 다크 테마 UI

한국투자증권 실시간 자동매매

KIS Open API로 실시간 시세를 받아 자동매매·주문 저널·손실 서킷브레이커까지 갖춘 트레이딩 웹앱

약 11,000줄 · 의존성 0

한국투자증권(KIS) Open API 기반 실시간 시세·자동매매 웹앱으로, npm 의존성 없이 Node 빌트인(net·https·crypto)만으로 RFC6455 WebSocket을 직접 구현해 실시간 체결을 SSE로 브라우저에 중계합니다. 주문·체결은 node:sqlite(WAL)에 저널링하고, 자동매매에는 손실 서킷브레이커로 리스크를 통제합니다.

핵심 기능

  • KIS 실시간 시세 수신 — RFC6455 WebSocket 자체 구현(net/https/crypto), WS→SSE 중계
  • 자동매매 엔진 + 손실 서킷브레이커(리스크 차단)
  • 주문·체결 저널(node:sqlite WAL) + 체결통보 정합성 검증
  • 인증·CORS 처리 프록시 서버 분리
  • 자체 테스트 하니스 5종(engine·realtime·fallback·journal·auth)
  • 전수조사 감사 워크플로(CRITICAL→LOW 등급별 발견·수정·검증)

개발 여정

개발 여정

두 프로젝트 모두 Phase 단위로 기능을 분리해 점진적으로 설계하고 커밋하며 완성했습니다. 각 Phase는 독립적으로 동작·검증 가능한 단위입니다.

실시간 협업 화이트보드

  1. 1

    캔버스 기반 · 도형 그리기

    Konva 캔버스 위에서 사각형·원 도형을 그리는 기본 구조를 구축했습니다.

  2. 2

    실시간 도형 추가 동기화 + 다크 테마

    shape:add 이벤트로 도형 추가를 모든 참여자에게 실시간 전파하고 다크 테마를 적용했습니다.

  3. 3

    이동 · 삭제 동기화

    shape:update·shape:delete 이벤트로 도형 이동과 삭제까지 동기화 범위를 확장했습니다.

  4. 4

    방(room) 격리 + 신규 접속 스냅샷 동기화

    URL 기반 방 단위로 세션을 격리하고, 신규 접속자에게 canvas:sync 스냅샷으로 전체 상태를 전달합니다.

  5. 5

    SQLite 상태 저장

    node:sqlite로 캔버스 상태를 영속화해 서버 재시작 후에도 데이터가 복원됩니다.

  6. 6

    실시간 커서 + 접속자(presence) + 재연결

    참여자 커서와 presence를 실시간 공유하고, 자동 재연결과 끊김 상태 배너를 추가했습니다.

공공기관 보안 영업 코파일럿 에이전트

  1. 0

    프로젝트 셋업 · API 키 관리

    venv, Anthropic SDK, .env 기반 키 관리와 Claude 호출 테스트로 기반을 마련했습니다.

  2. 1

    나라장터 공고 수집

    나라장터 OpenAPI로 입찰공고를 수집·파싱하는 파이프라인을 구축했습니다.

  3. 2

    보안 분류 (Claude)

    Claude로 보안/비보안 여부와 카테고리를 자동 분류합니다.

  4. 3

    제품 매칭

    분류된 공고를 내부 제품 카탈로그와 매칭하고 근거를 제시합니다.

  5. 4

    에이전트 (Tool Use)

    Claude Tool Use 패턴으로 분류·매칭을 수행하는 에이전트를 구성했습니다.

  6. 5

    영업 토킹포인트 생성 + 우선순위 스코어링

    고객 요구→강점→차별점 토킹포인트를 생성하고, 적합도·카테고리 가중·마감 임박도로 결정론적 우선순위를 산정합니다.

  7. 6

    Streamlit 대시보드

    필터·검색·CSV 내보내기를 갖춘 Streamlit 대시보드와 무료 경로 실행 모드를 구축했습니다.

  8. 7

    RFP PDF 분석 · 일일 브리핑 자동화

    RFP PDF에서 요구사항을 추출하고, 중복 제거된 신규 보안공고 일일 브리핑을 스케줄러로 자동화했습니다.
